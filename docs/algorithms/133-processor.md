# ABT-1.3.3-Processor: 명확한 지시사항 제공 알고리즘 v1.2

이 알고리즘은 지침 1.3.3(감각적 특징)을 준수하며, 모양, 크기, 위치, 소리 등 감각적 정보에만 의존하여 안내하고 있는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Text Analysis:** `aria-label`, `alt`, 일반 텍스트 노드 콘텐츠.
- **Visual Data:** 요소의 CSS 스타일(색상, 형태, 크기), 위치 정보.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 감각 지시어 키워드 매칭
- **위치 지시어:** "왼쪽", "오른쪽", "위쪽", "아래쪽", "상단", "하단", "옆", "앞", "뒤".
- **형태/크기 지시어:** "둥근", "네모난", "큰", "작은", "굵은", "얇은", "원형", "사각형".
- **색상 지시어:** "빨간", "파란", "초록", "노란", "어두운", "밝은".
- **매칭 정교화 로직:** 1글자 키워드(예: "위", "옆", "큰")는 독립적인 단어로 쓰였을 때(앞뒤 공백 또는 문장 부호 존재)만 매칭하여 "위치", "위원회", "위험" 등의 일반 단어에서 발생하는 오탐지(False Positive)를 방지한다.
- **지시사항 결합 필터링:** "왼쪽", "오른쪽" 등의 감각 지시어만으로는 부족하며, "버튼", "링크", "클릭", "참조" 등 실제 지시/안내 성격을 띠는 키워드와 함께 쓰였을 때만 진단 대상으로 분류한다.
### [단계 B] 문맥 분석 (Contextual Analysis)
- **Rule 1.1:** 키워드가 "버튼", "아이콘", "클릭", "누르기", "참고" 등의 행동 유도어와 결합되어 있는가?
  - 예: "오른쪽 버튼을 누르세요", "빨간색은 필수 항목입니다".
  - 결과: **[검토 필요]** (대체 수단 없는 감각 특징 의존 가능성)

### [단계 C] 대체 수단 존재 여부 검사
- **Rule 2.1:** 색상 지시어 사용 시, 텍스트(예: "필수")가 병기되어 있는가?
  - 결과: **[적절]**
  - 미존재 시: **[부적절]**

## 3. 최종 상태 정의 (Final Status)
1. **오류:** (기계적으로 판단하기 어려우나 명백한 경우)
2. **부적절:** 색상이나 위치 정보만으로 기능을 안내하고 텍스트 대안이 없는 경우.
3. **수정 권고:** 지시어 사용이 모호하여 사용자 혼란을 초래할 수 있는 경우.
4. **검토 필요:** 감각 지시어가 포함된 안내 문구 중, 대체 수단의 존재 여부 및 명확성을 기계적으로 확정하기 어려운 경우. 전문가가 해당 문구와 UI 요소를 확인하여 특정 감각에만 의존하지 않는지 판단해야 함.
5. **적절:** 감각적 특징과 함께 논리적인 텍스트(이름, 역할 등)가 제공됨.
