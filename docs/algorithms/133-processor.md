# ABT-1.3.3-Processor: 명확한 지시사항 제공 v0.13

### 🔗 References
- KWCAG 2.2: 1.3.3 명확한 지시사항 제공
- WCAG 2.2: 1.3.3 Sensory Characteristics (A) (Reference)

---

이 알고리즘은 **KWCAG 1.3.3(명확한 지시사항 제공)** 지침을 준수하며, 정보나 조작 방법을 전달할 때 오직 시각적 형태, 크기, 위치 또는 소리 등 감각적 특징에만 의존하는 지시사항이 있는지 진단한다. 단순 단어 포함 여부가 아닌, 감각 표현과 지시성 키워드가 결합된 **구문 패턴(Phrase Pattern)**을 탐지하여 오탐지를 최소화한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Text Analysis:** 요소 내의 직접적인 텍스트 노드 콘텐츠 추출.
- **Pattern Repository:** 감각적 특징(위치, 색상, 형태, 소리)과 조작/안내 키워드가 결합된 정규표현식 패턴 풀 활용.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 지능형 구문 패턴 매칭 (Pattern Matching)
- **핵심 원칙:** 단일 키워드(예: "위", "음성")만으로는 진단하지 않으며, 반드시 안내의 성격을 띠는 단어와 결합된 경우만 포착한다.
- **탐지 패턴 예시:**
  - **위치+지시:** "왼쪽/오른쪽/상단/하단" + "버튼/메뉴/항목/클릭/참조"
  - **색상+지시:** "빨간/파란/초록/어두운" + "색/버튼/아이콘/항목"
  - **형태+지시:** "동그란/네모난/원형/사각형" + "아이콘/버튼/모양"
  - **소리+지시:** "소리/신호음/비프/음성" + "로/를/가/이/들리면/확인"

### [단계 B] 오탐지 방지 로직 (False Positive Reduction)
- **독립 단어 검증:** 1글자 또는 짧은 단어의 경우 앞뒤 문맥(공백, 부호)을 확인하여 일반 명사에 포함된 경우(예: "위치", "위원회")를 제외한다.
- **키워드 복합 검증:** "음성"과 같은 단어는 기능 명칭(예: 음성 검색)에서 흔히 쓰이므로, "음성...확인"과 같이 실제 지시사항의 맥락이 형성될 때만 선별적으로 탐지한다.

## 3. 최종 상태 정의 (Final Status)
1. **검토 필요:** 감각 표현과 지시어가 결합된 의심 구문이 발견된 경우. 전문가가 해당 지시사항에 텍스트 대안(예: 이름, 역할)이 병행 제공되는지 확인해야 함.
2. **부적절:** 텍스트 대안 없이 오직 시각/청각적 특징만으로 조작을 유도하는 것이 명백한 경우.
3. **적절:** 감각적 특징을 언급하더라도 논리적인 이름(예: "왼쪽 [로그인] 버튼")을 함께 제공하여 정보 전달이 명확함.
4. **N/A:** 페이지 내에 지시/안내 성격의 텍스트 콘텐츠가 존재하지 않는 경우.
