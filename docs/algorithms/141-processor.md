# ABT-1.4.1-Processor: 색에 무관한 콘텐츠 인식 v0.11

### 🔗 References
- KWCAG 2.2: 1.4.1 색에 무관한 콘텐츠 인식
- WCAG 2.2: 1.4.1 Use of Color (A) (Reference)

---

이 알고리즘은 지침 1.4.1(색에 무관한 콘텐츠 인식)을 준수하며, 정보의 전달, 동작의 지시, 응답의 유도 또는 시각적 요소의 구분이 오직 색상만으로 이루어지지 않는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Visual Styles:** 요소의 `color`, `background-color`, `text-decoration`.
- **Graphic Elements:** `<canvas>`, `<svg>`, 그리고 차트/그래프 관련 클래스나 ID를 가진 컨테이너.
- **Content:** 요소 내 텍스트, 밑줄 존재 여부.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 링크 구분 검사 (Link Distinction)
- **Rule 1.1:** 본문 내 링크(`<a>`)가 주변 텍스트와 오직 색상만으로 구분되는가?
  - 조건: `text-decoration`이 `underline`이 아니면서 주변 텍스트와 색상만 다른 경우.
  - 결과: **[검토 필요]**.

### [단계 B] 시각적 데이터 콘텐츠 검사 (Graphic Data)
- **Rule 2.1:** 그래프, 차트 등 복잡한 정보를 전달하는 그래픽 요소가 존재하는가?
  - 대상: `<canvas>`, `<svg>`, `.chart`, `.graph` 등.
  - 분석: 해당 요소들이 색상만으로 데이터 계열을 구분하고 있는지 육안 검토가 필요함.
  - 결과: **[검토 필요]**.

### [단계 C] 배경색 의존성 검사
- **Rule 3.1:** 텍스트나 레이블 없이 배경색만으로 상태나 정보를 전달하는 요소가 있는가?
  - 결과: **[검토 필요]**.

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 색상만으로 정보를 전달하는 것이 명백한 경우.
2. **부적절:** 색상 외의 시각적 장치가 미흡한 경우.
3. **검토 필요:** 그래프/차트 콘텐츠, 밑줄 없는 링크 등 전문가의 정성적 판단이 필요한 경우.
4. **수정 권고:** 색상과 함께 다른 장치가 있으나 인지도가 낮은 경우.
5. **적절:** 색상 외에도 형태, 패턴, 텍스트 등을 통해 정보를 명확히 중복 전달함.
