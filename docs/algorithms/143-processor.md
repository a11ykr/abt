# ABT-1.4.3-Processor: 텍스트 콘텐츠의 명도 대비 v0.12

### 🔗 References
- KWCAG 2.2: 1.4.3 텍스트 콘텐츠의 명도 대비
- WCAG 2.2: 1.4.3 Contrast (Minimum) (AA) (Reference)

---

이 알고리즘은 **KWCAG 1.4.3(텍스트 콘텐츠의 명도 대비)** 지침을 준수하며, 웹 페이지의 모든 텍스트 콘텐츠가 배경색과 충분한 대비를 이루어 저시력 사용자나 고령자도 정보를 명확히 인지할 수 있는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Text Attributes:** 글자색(`color`), 배경색(`background-color`), 폰트 크기(`font-size`), 굵기(`font-weight`).
- **Computed Styles:** 부모 요소로부터 상속되거나 겹쳐진 실제 렌더링 색상 추출.
- **Exceptions:** 이미지 대체 기법(IR), 장식적 요소, 비활성 요소 감지.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 텍스트 크기별 기준 설정
- **Rule 1.1 (일반 텍스트):** 18pt(24px) 미만 또는 굵은 14pt(18.7px) 미만 텍스트.
  - 최소 대비 기준: **4.5 : 1**
- **Rule 1.2 (큰 텍스트):** 18pt(24px) 이상 또는 굵은 14pt(18.7px) 이상 텍스트.
  - 최소 대비 기준: **3 : 1**

### [단계 B] 실시간 명암비 계산
- **Formula:** WCAG 2.x 상대 휘도(Luminance) 공식을 사용하여 대비값을 산출한다.
- **Deduction:** 기준 미달 시 항목당 감점 모델을 적용하여 지침 점수를 계산한다.

### [단계 C] 특수 상황 처리 및 예외
- **Rule 3.1 (배경 이미지/그라데이션):** 배경이 단색이 아닌 경우.
  - 결과: **[검토 필요]** (가장 가독성이 낮은 지점 확인 필요).
- **Rule 3.2 (IR 기법):** 시각적으로 숨겨진 텍스트.
  - 결과: **[검사 제외]** (명도 대비 준수 대상이 아님).

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 텍스트 명암비가 기준(4.5:1 또는 3:1)에 미달함.
2. **부적절:** 가독성이 현저히 떨어지거나 브랜드 색상과 배경의 충돌이 심한 경우.
3. **수정 권고:** 기준은 통과했으나 더 높은 대비(7:1 등)가 권장되는 중요한 정보.
4. **검토 필요:** 반투명 배경, 동적 색상 변화 등 기계적 측정이 불충분한 경우.
5. **적절:** 모든 텍스트 콘텐츠가 표준 명암비 기준을 충족함.
