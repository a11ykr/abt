# ABT-1.4.4-Processor: 콘텐츠 간의 구분 v0.11

### 🔗 References
- KWCAG 2.2: 1.4.4 콘텐츠 간의 구분
- WCAG 2.2: N/A (KWCAG 전용 지침, 정보 및 관계(1.3.1)와 일부 연관)

---

이 알고리즘은 **KWCAG 1.4.4(콘텐츠 간의 구분)** 지침을 준수하며, 웹 페이지의 이웃한 콘텐츠가 테두리, 구분선, 배경색 명도 차, 간격 등을 통해 시각적으로 명확하게 구분되는지 진단한다. 특히 입력 서식과 버튼, 주요 레이아웃 블록 간의 독립성을 중점적으로 분석한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Visual Styles:** 요소의 `border`, `background-color`, `box-shadow`.
- **Spacing:** 요소 간의 거리(`margin`, `padding`) 및 절대적 좌표 차이.
- **Form Controls:** `input`, `select`, `textarea`, `button` 요소의 시각적 경계 정보.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 입력 서식 구분 검사 (Form Control Distinction)
- **Rule 1.1:** 입력 필드(`input`, `textarea` 등)가 배경과 시각적으로 구분되는 테두리나 배경색 차이를 가지고 있는가?
  - 결과: 테두리도 없고 배경색도 부모와 동일한 경우 **[수정 권고]**.

### [단계 B] 버튼 및 컨트롤 간격 검사
- **Rule 2.1:** 인접한 버튼이나 클릭 가능한 요소들 사이에 충분한 여백이 있거나 시각적 구분선이 존재하는가?
  - 결과: 물리적 간격이 매우 좁고 시각적 구분 장치가 없는 경우 **[검토 필요]**.

### [단계 C] 레이아웃 블록 구분 검사 (Layout Block Differentiation)
- **Rule 3.1:** 헤더, 본문, 사이드바 등 주요 영역이 시각적으로 독립된 블록으로 인지되는가?
  - 결과: 시각적 장치(테두리, 색상 차 등)를 통한 구분이 모호한 경우 **[검토 필요]**.

## 3. 최종 상태 정의 (Final Status)
1. **부적절:** 콘텐츠 간의 경계가 전혀 없어 정보의 범위를 파악하기 힘든 경우.
2. **수정 권고:** 입력 필드 등 중요 컨트롤의 시각적 경계가 약한 경우.
3. **검토 필요:** 레이아웃 영역이나 인접 버튼 간의 시각적 구분이 전문가의 정성적 확인이 필요한 수준인 경우.
4. **적절:** 모든 이웃한 콘텐츠가 다양한 시각적 단서를 통해 명확히 구분됨.
5. **N/A:** 페이지 내에 구분할 대상 콘텐츠가 존재하지 않는 경우.
