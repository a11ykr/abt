# ABT-2.1.3-Processor: 조작 가능 v0.11

### 🔗 References
- KWCAG 2.2: 2.1.3 조작 가능
- WCAG 2.2: 2.5.5 Target Size (Enhanced) (AAA) (Reference)

---

이 알고리즘은 **KWCAG 2.1.3(조작 가능)** 지침을 준수하며, 터치 디바이스나 마우스 사용 시 오조작을 방지하기 위해 대화형 요소가 충분한 크기(최소 24x24px)를 갖추고 있는지 진단한다.
## 1. 데이터 수집 단계 (Data Collection)
- **Target Elements:** 모든 대화형 요소 (`a`, `button`, `input`, `select`, `[role="button"]` 등).
- **Physical Metrics:** `getBoundingClientRect()` 기반의 가로/세로 크기 및 좌표.
- **Visual Context:** 인라인 텍스트 내 포함 여부, 주변 요소와의 거리.
## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 컨트롤 크기 및 간격 검사 (Target Size)
- **Rule 1.1 (최소 크기):** 
  - 대상 요소의 크기가 **24x24px 미만**인 경우.
  - **특수 예외:** 문장 중간의 링크(Inline Link)는 텍스트 가독성을 저해할 수 있으므로 진단에서 제외함.
  - 결과: **[수정 권고]**.
- **Rule 1.2 (컨트롤 밀집):** 
  - 인접한 대화형 요소 사이의 간격이 **8px 미만**이면서 각 요소의 크기도 24px 이하인 경우.
  - 결과: **[부적절]** (오조작 위험 높음).

## 3. 최종 상태 정의 (Final Status)
1. **부적절:** 좁은 간격과 작은 크기가 결합되어 오조작 유발 가능성이 큰 구간.
2. **검토 필요:** 최소 권장 크기(24x24px) 미달 요소 (터치 환경을 고려한 수동 확인 유도).
3. **적절:** 요소가 충분한 크기를 가짐 (인라인 텍스트 링크 등 예외 포함).

> **참고:** 복잡한 포인터 제스처 대안은 **2.5.1(단일 포인터 입력 지원)**에서, 레이블 불일치 이슈는 **2.5.3(레이블과 네임)**에서 별도로 다룹니다.
