# ABT-2.1.3-Processor: 조작 가능 v0.10

### 🔗 References
- KWCAG 2.2: 2.1.3 조작 가능
- WCAG 2.2: 2.5.5 Target Size (Enhanced) (AAA) (Reference)

---

이 알고리즘은 **KWCAG 6.1.3(조작 가능)** 지침을 준수하며, 모든 컨트롤은 오조작 없이 조작할 수 있도록 충분한 크기와 간격을 확보하고 있는지, 복잡한 포인터 제스처의 대안이 있는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Target Elements:** 모든 대화형 요소 (`a`, `button`, `input`, `select`, `[role="button"]` 등).
- **Physical Metrics:** `getBoundingClientRect()` 기반의 가로/세로 크기 및 좌표.
- **Visual Context:** 인라인 텍스트 내 포함 여부, 주변 요소와의 거리.
- **Gesture Handlers:** `dragstart`, `touchmove`, `pointermove` 등 복잡한 궤적 기반 이벤트 리스너 감지.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 컨트롤 크기 및 간격 검사 (Target Size)
- **Rule 1.1 (최소 크기):** 
  - 대상 요소의 크기가 **24x24px 미만**인 경우.
  - **특수 예외:** 문장 중간의 링크(Inline Link)는 텍스트 가독성을 저해할 수 있으므로 진단에서 제외함.
  - 결과: **[수정 권고]**.
- **Rule 1.2 (컨트롤 밀집):** 
  - 인접한 대화형 요소 사이의 간격이 **8px 미만**이면서 각 요소의 크기도 24px 이하인 경우.
  - 결과: **[부적절]** (오조작 위험 높음).

### [단계 B] 복잡한 포인터 제스처 (Pointer Gestures)
- **Rule 2.1 (제스처 감지):** 드래그 앤 드롭, 슬라이드 잠금 해제 등 단순 클릭이 아닌 복잡한 궤적 기반 조작이 감지되는 경우.
  - 결과: 무조건 **[검토 필요]**.
  - 가이드: "복잡한 포인터 제스처가 감지되었습니다. 드래그 대신 단순 클릭만으로도 동일한 기능을 수행할 수 있는 대안이 제공되는지 확인하십시오."

### [단계 C] 레이블과 성명 일치 (Label in Name)
- **Rule 3.1:** 시각적 텍스트 레이블이 `aria-label` 등의 성명에 포함되지 않은 경우.
  - 결과: **[오류]** (음성 인식 조작 시 문제 발생).

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 시각적 레이블과 접근성 성명 불일치, 조작 불가능한 겹침 현상.
2. **부적절:** 좁은 간격과 작은 크기가 결합되어 오조작 유발 가능성이 큰 구간.
3. **수정 권고:** 최소 권장 크기(24px) 미달 요소 (인라인 링크 제외).
4. **검토 필요:** 복잡한 제스처 요구 기능, 기계적 측정이 불가능한 커스텀 위젯.
5. **적절:** 충분한 크기와 간격 확보, 단일 포인터 클릭 기반의 명확한 조작성 제공.
