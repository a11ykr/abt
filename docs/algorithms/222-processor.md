# ABT-2.2.2-Processor: 정지 기능 제공 v0.11

### 🔗 References
- KWCAG 2.2: 2.2.2 정지 기능 제공
- WCAG 2.2: 2.2.2 Pause, Stop, Hide (A) (Reference)

---

이 알고리즘은 **KWCAG 2.2.2(정지 기능 제공)** 지침을 준수하며, 자동으로 변경되는 콘텐츠(슬라이드, 롤링 등)를 사용자가 제어할 수 있는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Moving Elements:** CSS 애니메이션(`infinite`), `setTimeout/setInterval` 기반 위치 이동 요소.
- **Auto-updating:** 정기적인 API 호출을 통한 콘텐츠 갱신(주가, 뉴스 등).
- **Control Buttons:** "정지", "일시정지", "Pause", "Stop", "||" 기호를 포함한 버튼.
- **Event Handlers:** `mouseenter`, `focus` 시 애니메이션 정지 로직 포함 여부.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 자동 변경 콘텐츠 탐지
- **Rule 1.1:** 슬라이더, 롤링 배너, `marquee` 등 자동으로 내용이 변할 가능성이 있는 요소를 탐지.
  - 조건: `class` 속성에 `carousel`, `slider`가 있거나 `role="marquee"`인 요소.
  - 결과: 요소가 존재하면 5초 지속 여부와 상관없이 **[검토 필요]** (정지 수단 확인 유도).

### [단계 B] 제어 수단 검증
- **Rule 2.1 (물리적 버튼):** 인접 영역에 명확한 정지 버튼이 있는가?
- **Rule 2.2 (인터랙션 정지 - 전문가 수동 검사):** 전문가가 해당 콘텐츠에 마우스를 올리거나 키보드로 초점(Focus)을 이동했을 때 움직임이 멈추는지 수동으로 확인해야 한다.
  - **ABT 엔진 역할**: 움직이는 요소를 찾아내고, 전문가에게 해당 요소에 대한 수동 테스트를 안내. 사용자의 동작 후 요소의 움직임 변화를 모니터링하여 보고.
  - 결과: 움직임이 멈추지 않으면 **[부적절]**.

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 5초 이상 자동 변경되는데 어떠한 제어 수단도 없는 경우.
2. **부적절:** 제어 수단은 있으나 키보드 접근이 불가하거나 인지하기 매우 어려운 경우.
3. **검토 필요:** 슬라이더나 캐러셀 등 동적 요소가 감지되었으나 기계적으로 정지 수단을 완벽히 확신할 수 없는 경우 (기본값).
4. **적절:** 명확한 제어 버튼과 함께 마우스 오버/초점 시 정지 기능이 제공됨.
