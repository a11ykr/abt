# ABT-2.3.1-Processor: 깜빡임과 번쩍임 사용 제한 v0.11

### 🔗 References
- KWCAG 2.2: 2.3.1 깜빡임과 번쩍임 사용 제한
- WCAG 2.2: 2.3.1 Three Flashes or Below Threshold (A) (Reference)

---

이 알고리즘은 **KWCAG 2.3.1(깜빡임과 번쩍임 사용 제한)** 지침을 준수하며, 광과민성 발작을 유발할 수 있는 '초당 3회 이상의 번쩍임'이 포함된 콘텐츠의 유무를 진단한다. 번쩍임의 정확한 빈도와 면적을 기계적으로 정밀 분석하는 것은 일반적인 확장 프로그램 환경에서 한계가 크므로, 이 프로세서는 **"가장 위험한 태그의 자동 탐지 및 전문가의 수동 검토 유도"**를 주 목적으로 한다.

## 1. 수동 검토 가이드 (What to Look For)
전문가는 페이지 내에서 다음과 같은 시각적 현상이 나타나는지 직접 확인해야 한다:
- **깜빡임(Blinking):** 동일한 색상의 화면이나 텍스트가 빠르게 켜지고 꺼짐을 반복하는 현상.
- **번쩍임(Flashing):** 전체 화면이나 특정 영역의 색상이 밝은 색과 어두운 색 사이에서 빠르게 교차하며 변하는 현상.
- **주요 대상:** 재생 중인 동영상, 화려한 광고 배너(GIF), 전체 화면 애니메이션.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 위험 태그 탐지 (Deprecated Elements)
- **Rule 1.1:** 폐기된 비표준 태그인 `<marquee>` 또는 `<blink>`가 사용되었는가?
  - 분석: 해당 태그들은 사용자 제어가 불가능한 상태에서 콘텐츠를 계속해서 깜빡이거나 움직이게 만들어 접근성을 크게 해친다.
  - 결과: 발견 즉시 **[오류]** 처리.

### [단계 B] 동적 미디어 요소 안내 (Media Check)
- **Rule 2.1:** 페이지 내에 `<video>` 등 동영상이 삽입되어 있는가?
  - 결과: 요소 발견 시 **[검토 필요]**.
  - 가이드: "영상 내에 1초에 3회 이상 번쩍이는 콘텐츠가 포함되어 있는지 수동으로 확인하세요."

### [단계 C] 전역 수동 검토
- **Rule 3.1:** 기계가 탐지하지 못한 CSS/JS 기반 애니메이션이나 동적 캔버스가 있을 수 있으므로, 페이지 전체를 대상으로 깜빡임 여부를 묻는 기본 리포트를 생성.
  - 결과: **[검토 필요]** (전문가의 육안 확인 후 '적절'로 판정 유도).

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 웹 표준에서 폐기된 `<marquee>`, `<blink>` 태그를 사용하여 강제적인 깜빡임을 유발함.
2. **검토 필요:** 동영상(`video`)이 존재하거나, 페이지 전반에 걸쳐 초당 3회 이상 번쩍이는 시각 요소가 있는지 전문가가 아직 확인하지 않음.
3. **적절:** 페이지 내에 깜빡이거나 번쩍이는 요소가 전혀 없거나, 3회 미만으로 안전한 수준임.
