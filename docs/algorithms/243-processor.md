# ABT-2.4.3-Processor: 적절한 링크 텍스트 v0.10

### 🔗 References
- KWCAG 2.2: 2.4.3 적절한 링크 텍스트
- WCAG 2.2: 2.4.4 Link Purpose (In Context) (A) (Reference)

---

이 알고리즘은 **KWCAG 6.4.3(적절한 링크 텍스트)** 지침을 준수하며, 링크의 텍스트가 해당 링크의 목적이나 대상(목적지)을 명확히 설명하고 있는지 진단한다. 텍스트가 불충분하더라도 프로그램적으로 연결된 맥락(Context)을 통해 목적을 파악할 수 있는지 분석한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Accessible Name:** 링크의 최종 계산된 이름 (`aria-label`, `aria-labelledby`, 텍스트 노드, 이미지 `alt`).
- **Programmatic Context:** 
  - 부모 요소(`p`, `li`, `td`, `dd` 등)의 텍스트.
  - `aria-describedby`로 연결된 요소의 텍스트.
  - 동일 목록(`ul/ol`) 내 인접한 제목 요소.
- **Destination:** `href` 속성 및 동일 페이지 내 중복 여부.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 링크 텍스트의 유효성 및 맥락 분석
- **Rule 1.1 (독립적 명확성):** 텍스트만으로 목적이 명확한가? (예: "공지사항 게시판으로 이동") -> **[적절]**.
- **Rule 1.2 (지시어 및 기호 - 맥락 의존성):** 
  - **유형 1 (단순 지시어):** "더 보기", "자세히", "여기" 등.
    - 주변 맥락(부모/인접 요소)에 구체적 정보가 있으면 **[검토 필요]**.
    - 맥락 정보조차 없으면 **[부적절]**.
  - **유형 2 (기호 및 공백):** ">", "+", 또는 비어있는 경우.
    - 이는 1.2의 연장선상에서 가장 심각한 정보 결여로 간주. 맥락이 있더라도 식별성이 낮으므로 **[부적절]** 또는 **[오류]**.
- **Rule 1.3 (URL 노출):** 의미 있는 설명 없이 URL(https://...)을 그대로 노출한 경우 **[수정 권고]**.

### [단계 B] 중복 및 관계 분석
- **Rule 2.1 (동일 이름, 다른 URL):**
  - 링크 이름은 같으나 목적지가 다른 경우.
  - 각 링크의 **'부모 맥락'**이 서로 다르면 **[적절]** (예: 뉴스 목록의 개별 '자세히 보기').
  - 맥락까지 동일한데 URL만 다르면 사용자를 혼란스럽게 하므로 **[부적절]**.
- **Rule 2.2 (인접 중복 링크):** 동일한 목적지를 가진 이미지와 텍스트 링크가 나란히 있는 경우 **[수정 권고]** (병합 권장).

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 링크 텍스트가 아예 없거나(공백), 이미지 링크에 대체 텍스트(`alt`)가 누락된 경우.
2. **부적절:** 
   - ">", "+" 등 기호로만 구성된 링크.
   - 맥락 정보 없이 "클릭", "여기" 등 지시어만 사용.
   - 맥락이 동일함에도 URL이 다른 중복 링크 이름.
3. **검토 필요:** "자세히 보기" 등 지시어를 사용했으나 주변 맥락 정보가 존재하는 경우 (전문가 확인).
4. **수정 권고:** 기계적 URL 노출, 인접한 동일 목적지 링크의 분리.
5. **적절:** 링크 이름 또는 제공된 맥락을 통해 목적을 명확히 알 수 있음.
