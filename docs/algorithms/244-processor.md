# ABT-2.4.4-Processor: 고정된 참조 위치 정보 v0.12

### 🔗 References
- KWCAG 2.2: 2.4.4 고정된 참조 위치 정보
- WCAG 2.2: 2.4.13 Page Break Locators (A) (Reference)

---

이 알고리즘은 **KWCAG 2.4.4(고정된 참조 위치 정보)** 지침을 준수하며, 온라인 문서가 페이지가 고정된 원본 매체(인쇄물, PDF 등)와 함께 제공될 때 페이지 번호 등의 참조 위치 정보가 일관되게 유지되는지 진단한다. 자동화 도구는 원본 매체의 존재 여부를 알 수 없으므로, 잠재적 참조 요소를 탐지하고 전문가에게 수동 검토 가이드를 제공하는 것을 목적으로 한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Reference Indicators:** 클래스명(`.page-number`, `.pg-num`)이나 ARIA 레이블에 '페이지' 또는 'page'가 포함된 요소 탐색.
- **Navigation Elements:** 페이지 이동과 관련된 내비게이션 바 또는 입력 필드 정보 수집.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 잠재적 참조 요소 탐지
- **Rule 1.1 (Page Number Detection):** 페이지 번호로 추정되는 속성이나 텍스트를 가진 요소가 있는가?
  - 결과: 발견 시 **[검토 필요]**.
  - 가이드: "페이지 번호 참조용으로 추정되는 요소가 탐지되었습니다. 페이지가 고정된 원본 매체(인쇄물, PDF 등)와 동일한 위치 정보를 제공하고 있는지 확인하세요."

### [단계 B] 전역 수동 검토 가이드 생성
- **Rule 2.1 (Manual Review):** 특정 요소가 탐지되지 않더라도, 문서 전체를 대상으로 지침 준수 여부 확인이 필요함.
  - 결과: 무조건 **[검토 필요]** 리포트 생성.
  - 가이드: "페이지 번호가 포함된 원본 매체와 함께 제공되는 문서인 경우, 해당 매체의 고정된 참조 위치를 온라인에서도 동일하게 제공하는지 확인하십시오."

## 3. 최종 상태 정의 (Final Status)
1. **검토 필요:** 원본 매체와의 일관성 확인이 전문가에 의해 아직 수행되지 않음.
2. **적절:** (전문가가 '검토 필요' 항목을 확인 후 일관성이 보장된다고 판정한 경우에만 적용)
3. **N/A:** 페이지 내에 참조할 만한 페이지 정보나 내비게이션이 없으며, 원본 매체와 무관한 순수 웹 콘텐츠인 경우.
