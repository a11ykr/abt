# ABT-2.5.1-Processor: 단일 포인터 입력 지원 v0.11

### 🔗 References
- KWCAG 2.2: 2.5.1 단일 포인터 입력 지원
- WCAG 2.2: 2.5.1 Pointer Gestures (A) (Reference)

---

이 알고리즘은 **KWCAG 2.5.1(단일 포인터 입력 지원)** 지침을 준수하며, 드래그, 핀치 줌, 스와이프 등 복잡한 제스처가 필요한 기능에 대해 단일 포인터(탭/클릭)만으로 조작 가능한 대체 수단이 제공되는지 진단한다. 브라우저 확장 프로그램의 기술적 제한(Isolated World)으로 인해 직접적인 이벤트 리스너 감지보다는 복잡한 조작이 예상되는 UI 패턴을 탐지하여 전문가에게 확인을 요청하는 방식을 취한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Potential Gesture Elements:** `draggable="true"`, `role="slider"`, `role="scrollbar"`, `<canvas>` 요소.
- **Library Patterns:** 클래스명이나 ID에 `carousel`, `slider`, `swiper`, `map` 등 제스처 기반 라이브러리 키워드가 포함된 요소.
- **Visual Context:** 제스처 의심 요소 인근(부모/형제)의 클릭 가능한 요소(`button`, `a`, `[role="button"]`) 존재 여부.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 제스처 사용 예상 영역 식별 (Pattern Matching)
- **Rule 1.1 (속성 및 태그 기반):** `draggable` 속성이나 슬라이더 역할(`role`)이 부여된 경우, 혹은 지도 등으로 자주 쓰이는 대형 `canvas` 요소인 경우.
  - 결과: **[검토 필요]**.
  - 가이드: "복잡한 제스처 사용이 예상되는 요소입니다. 만약 해당 기능이 드래그나 스와이프를 요구한다면, 클릭만으로도 작동하는 대체 수단이 있는지 확인하세요."

### [단계 B] 대체 조작 수단 탐색
- **Rule 2.1 (보조 컨트롤 검색):** 제스처가 예상되는 영역 주변에 '이전/다음', '확대/축소' 등의 텍스트나 아이콘을 가진 버튼이 존재하는가?
  - 결과: 버튼 발견 시에도 **[검토 필요]** (해당 버튼이 실제로 제스처 기능을 완벽히 대체하는지 전문가 확인 필수).

## 3. 최종 상태 정의 (Final Status)
1. **검토 필요:** (기본 상태) 복잡한 제스처 사용이 의심되는 영역. 전문가가 실제 제스처 사용 여부와 단일 클릭 대안 존재 여부를 최종 판정함.
2. **적절:** (전문가가 '검토 필요' 항목을 확인 후 대안 수단이 충분하다고 판정한 경우)
3. **부적절:** 제스처가 필수적인 기능임에도 주변에 대체 가능한 버튼이 전혀 없는 경우 (전문가 판정에 따름).
4. **N/A:** 페이지 내에 제스처 기반 조작을 수행할 것으로 의심되는 요소가 전혀 없는 경우.
