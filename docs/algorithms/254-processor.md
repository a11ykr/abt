# ABT-2.5.4-Processor: 동작기반 작동 v0.20

### 🔗 References
- KWCAG 2.2: 2.5.4 동작기반 작동
- WCAG 2.2: 2.5.4 Motion Actuation (A) (Reference)

---

이 지침은 모바일 기기 등을 흔들거나 기울이는 동작 센서를 이용한 기능이 일반적인 인터랙션(클릭, 탭 등)으로도 수행 가능한지 진단합니다.

## 1. 진단 한계 및 수동 검사 전환 사유
브라우저 확장 프로그램(Content Script) 환경에서는 개발자가 의도한 모바일 기기 센서(`devicemotion`, `deviceorientation` 등)의 실제 동작 여부와 그에 대응하는 UI 버튼의 존재 여부를 100% 기계적으로 매칭할 수 없습니다. 
기존의 단순 키워드 스크립트 스캐닝 방식은 수많은 오탐(False Positive/Negative)을 발생시켜 진단 신뢰도를 떨어뜨리므로, 해당 항목은 **"전면 수동 검사(Pure Manual Check)"** 항목으로 전환되었습니다.

## 2. 분석 및 분류 로직 (Analysis Pipeline)
DOM을 탐색하지 않고, 페이지 스캔 시 즉시 **단일 수동 검사 리포트**를 생성하여 대시보드에 띄웁니다.

- **Rule 1.1 (수동 검사 안내):**
  - 결과: 항상 **[검토 필요]** 1건 발생.
  - 가이드: "[수동 검사 항목] 이 페이지(또는 앱)에서 기기를 흔들거나 기울이는 동작을 요구하는 기능이 있다면, 버튼 클릭 등 정적인 입력으로도 동일한 기능을 수행할 수 있는지 수동으로 점검하세요."

## 3. 최종 상태 정의 (Final Status)
1. **검토 필요 (Needs Review):** 
   - 진단 엔진이 스캔할 때마다 항상 출력되며, 진단자가 직접 앱의 기획서나 모바일 화면을 흔들어보며 대체 수단이 있는지 점검해야 함을 알립니다.
