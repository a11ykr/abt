# ABT-7.2.2-Processor: 찾기 쉬운 도움 정보 알고리즘 v1.2

### 🔗 KWCAG 2.2 Reference
- **7.2.2 (찾기 쉬운 도움 정보)**: 도움 정보가 제공되는 경우, 각 페이지에서 동일한 상대적인 순서로 접근할 수 있어야 한다.

---

이 알고리즘은 **KWCAG 7.2.2(찾기 쉬운 도움 정보)** 지침을 준수하며, 웹 사이트 내에서 사용자가 문제 해결을 위해 제공받는 연락처, FAQ, 도움말, 채팅 봇 등의 요소가 여러 페이지에 걸쳐 **시각적인 위치뿐만 아니라 마크업 순서 모두 일관성 있게 구현**되었는지 진단한다.
> 💡 **단일 페이지 검사 원칙 및 ABT 엔진 역할**
> ABT 엔진(브라우저 확장 프로그램)은 현재 로드된 단일 페이지의 DOM만을 분석하는 것을 원칙으로 한다. 따라서 다중 페이지 간의 일관성 검사는 엔진의 자동화 범위를 벗어나며, **탐지된 도움 정보 요소에 대해 전문가의 수동 검사를 강하게 유도**하는 방향으로 동작한다.

## 1. 대상 식별 및 데이터 수집 (ABT 엔진 역할)

### [대상 요소 추출]
- 페이지 내 반복적으로 제공되는 다음 유형의 사용자 도움 정보 식별:
  1. **담당자 상세 연락처**: 전화번호, 이메일, 주소, 운영시간 등
  2. **담당자 연락 방법**: 메신저, 채팅창, 게시판, SNS 등
  3. **도움말 옵션**: FAQ, 사용법 등
  4. **자동화된 연결 방법**: 챗봇 링크 또는 버튼 등

### [데이터 속성 파악]
- 추출된 도움 정보 요소의 DOM 트리 내 마크업 순서(XPath 또는 CSS Selector 패턴).
- 추출된 도움 정보 요소의 시각적 렌더링 좌표(Bounding Box)와 화면 내 위치(예: 하단 footer, 우측 하단 고정).
- 현재 페이지 URL과 요소의 메타데이터를 포함한 스냅샷 데이터를 추출하여 전문가에게 제시.

## 2. 분석 및 분류 로직 (단일 페이지 분석 및 전문가 가이드)

### [단계 A] 도움 정보 요소 탐지 및 스냅샷 생성
- **ABT 엔진 역할**: 현재 페이지 내에서 도움 정보 역할을 하는 요소를 탐지하고, 그 요소의 DOM 경로, 시각적 좌표, 현재 URL 등을 포함한 "스냅샷 데이터"를 생성하여 전문가에게 제시한다.
- 결과: 도움 정보가 탐지되면, 자동 검사로는 다중 페이지 일관성을 판단할 수 없으므로 **[검토 필요]** 상태로 전환.

### [단계 B] 전문가 수동 검사 가이드라인
도구는 전문가에게 다음 체크리스트를 제공하여, 여러 페이지를 탐색하며 도움 정보의 일관성을 수동으로 검증하도록 유도한다.
- *체크리스트*:
  - **시각적 위치 일관성**: 다른 페이지에서도 동일한 유형의 도움 정보가 유사한 시각적 위치에 나타나는가?
  - **마크업 순서 일관성**: 다른 페이지에서도 도움 정보 요소의 DOM 트리 내 상대적 순서가 동일하게 유지되는가?
  - **제공 방식 일관성**: 도움 정보의 제공 방식(텍스트 링크, 아이콘, 챗봇 위젯 등)이 페이지마다 일관적인가?

## 3. 최종 상태 정의 (Final Status)
1. **부적절 (Warning):**
   - 전문가가 수동 검사한 결과, 도움 정보(전화번호, 이메일, 챗봇 등)의 **시각적인 위치**나 **마크업 순서**가 페이지별로 확연히 다르게 나타나는 경우.
2. **검토 필요 (Review) [Default State for detection]:**
   - ABT 엔진이 현재 페이지에서 도움 정보 요소를 탐지했으며, 다중 페이지 간 일관성 여부를 전문가가 직접 확인해야 하는 상태. (탐지된 요소의 스냅샷 데이터를 제공하여 전문가의 판단 보조)
3. **적절 (Pass):**
   - 전문가가 수동 검사한 결과, 반복되는 도움 정보가 웹 사이트 내 동일 위치(시각적 위치 및 마크업 순서 모두)에 일관성 있게 구현됨이 확인된 경우.
4. **N/A:**
   - 현재 페이지에 도움 정보 요소가 존재하지 않음.
