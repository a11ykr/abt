# ABT-3.2.2-Processor: 찾기 쉬운 도움 정보 v0.20

### 🔗 References
- KWCAG 2.2: 3.2.2 찾기 쉬운 도움 정보
- WCAG 2.2: 3.2.6 Findable Help (A) (Reference)

---

이 지침은 사용자가 문제 해결을 위해 이용하는 도움 정보(FAQ, 고객센터, 챗봇 등)가 **여러 페이지에 걸쳐 시각적인 위치뿐만 아니라 마크업 순서까지 일관성 있게 구현**되었는지 진단합니다.

> 💡 **진단 한계 및 수동 검사 전환 사유**
> 단일 페이지 스캐너인 확장 프로그램 환경에서는 "이 페이지의 챗봇 위치가 다른 페이지의 챗봇 위치와 같은가?"라는 교차 검증을 기계적으로 수행할 수 없습니다. 따라서 현재 페이지 내에서 도움 정보(키워드)를 탐지하여 힌트를 제공하고, 최종 일관성 평가는 **전문가의 수동 검사**에 온전히 의존하는 방식으로 설계되었습니다.

## 1. 대상 식별 및 데이터 수집

### [대상 요소 추출]
- `a`, `button`, `[role="button"]`, `[role="link"]` 요소 중 다음 키워드를 포함하는 요소:
  - 한글: `faq`, `고객센터`, `도움말`, `챗봇`, `채팅`, `문의`, `연락처`
  - 영문: `help`, `contact`, `support`, `chat`

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 키워드 기반 도움 정보 탐지
- **Rule 1.1 (도움말 키워드 탐지):** 대상 요소의 텍스트(`innerText`), `aria-label`, `title`에 위 키워드가 포함되어 있는가?
  - 결과: 하나라도 감지될 경우 발견된 요소 갯수를 요약하여 **[검토 필요]** 리포트 단일 생성.
  - 가이드: "[수동 검사 안내] 현재 페이지에서 도움말 관련 요소가 탐지되었습니다. 다른 페이지들에서도 이 요소가 동일한 순서와 위치에 일관되게 제공되는지 수동으로 확인하세요."

### [단계 B] 미탐지 시 처리
- **Rule 1.2 (키워드 없음):** 어떠한 도움말 관련 키워드도 찾지 못한 경우.
  - 결과: **[N/A]** 상태로 리포팅.
  - 가이드: "명시적인 도움 정보 키워드가 탐지되지 않았습니다. 만약 텍스트 없이 아이콘으로만 제공된 챗봇 등이 있다면 육안으로 위치 일관성을 점검하세요."

## 3. 최종 상태 정의 (Final Status)

1. **검토 필요 (Needs Review):**
   - 페이지 내에서 고객센터, FAQ, 챗봇 등의 링크/버튼이 발견되었음. 진단자가 직접 웹사이트의 다른 페이지(서브 메뉴 등)를 돌아다니며 해당 요소가 똑같은 위치(예: 화면 우측 하단 고정 플로팅, Footer 영역 등)에 존재하는지 수동 점검해야 함.
2. **N/A:**
   - 엔진이 식별할 수 있는 전형적인 텍스트 기반의 도움 정보가 현재 페이지에 없음.
