# ABT-3.3.3-Processor: 접근 가능한 인증 v0.10

### 🔗 References
- KWCAG 2.2: 3.3.3 접근 가능한 인증
- WCAG 2.2: 3.3.7 Accessible Authentication (Minimum) (AA) (Reference)

---

이 알고리즘은 **KWCAG 3.3.3(접근 가능한 인증)** 지침을 준수하며, 사용자 인증 과정(로그인, 본인 확인 등)이 비밀번호 기억, 복잡한 캡차(CAPTCHA) 풀이, 퍼즐 맞추기 등 인지 기능에 과도하게 의존하는 경우를 탐지하고, 인지 테스트에 의존하지 않는 대체 인증 수단이 제공되는지 진단한다.

## 1. 대상 식별 및 데이터 수집 (Data Collection)

### [대상 요소 추출]
- **인증 관련 폼/요소**: 로그인 폼 (`<form action="/login">`), 회원가입 폼, 비밀번호 재설정 폼, CAPTCHA 영역, 2단계 인증(MFA) 관련 UI.
- **인지 테스트 유형 키워드**:
  - 비밀번호 입력 필드 (`<input type="password">`)
  - CAPTCHA (`<img src="captcha.jpg">`, `aria-label="보안문자"`, reCAPTCHA 스크립트 등)
  - 퍼즐/그림 맞추기 (`canvas` 요소, `aria-label`에 "퍼즐", "그림 맞추기" 등)
  - 복잡한 질문 답변 (예: "가장 친한 친구의 이름은?")
- **대체 인증 수단 키워드**:
  - 생체 인증 (`face ID`, `fingerprint`, `biometric`)
  - USB/보안 토큰 (`OTP`, `token`)
  - 기기 인증 (`device auth`)
  - 복사/붙여넣기 (`copy`, `paste` 활성화 여부)
  - 간편 인증 (`카카오톡 인증`, `네이버 인증`, `SNS 로그인`, `OAuth`)
  - 이름/전화번호 등 개인정보 확인 (`본인인증`, `personal info`)

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 인지 기능 테스트 요소 탐지
- **Rule 1.1 (비밀번호):** `<input type="password">` 필드가 존재하는가? (단독 사용 시 인지 테스트로 간주)
- **Rule 1.2 (CAPTCHA/퍼즐):** CAPTCHA 이미지, 퍼즐 요소, 혹은 관련 텍스트(예: "로봇이 아닙니다")가 탐지되는가?
- **Rule 1.3 (복잡한 질문):** 사용자가 직접 답변을 기억해야 하는 유형의 질문 폼이 존재하는가?
- 결과: 위 인지 기능 테스트 요소 중 하나라도 탐지되면 **[검토 필요]** 상태로 전환.

### [단계 B] 대체 인증 수단 탐지 한계 및 수동 검사 유도
- 정적 분석만으로는 페이지 내에 존재하는 복잡한 외부 인증(SNS 로그인 연동 등)이 "현재 캡차(인지 테스트)에 대한 유효한 대체 수단"인지 문맥을 판단하기 어렵습니다.
- 따라서, 비밀번호 필드가 발견되면 엔진은 패스워드 매니저 방해 여부만 기계적으로 판단(오류 처리)하고, 캡차 등 나머지 인지 테스트에 대한 검증은 전문가에게 가이드를 제공하여 **[검토 필요]** 상태로 넘깁니다.

### [단계 C] 전문가 수동 검사 유도 및 가이드 (ABT 엔진 역할)
- ABT 엔진은 인지 기능 테스트 요소와 탐지된 대체 수단 목록을 제공한다.
- 전문가는 다음 체크리스트를 바탕으로 최종 판정을 내린다:
  - "인지 기능 테스트(예: 복잡한 비밀번호 기억)가 필수적인 인증 수단으로 사용되는가?"
  - "인지 테스트에 의존하지 않는 **최소 한 가지 이상의 대체 인증 수단**이 제공되는가?"
  - "비밀번호 필드에 복사/붙여넣기 기능이 비활성화되어 있지 않은가?"

## 3. 최종 상태 정의 (Final Status)
1. **부적절 (Warning):**
   - 인지 기능 테스트에 과도하게 의존하는 인증 방식(예: 복잡한 비밀번호 기억, 복잡한 캡차)만 제공되고, 인지 테스트에 의존하지 않는 대체 인증 수단이 전혀 없는 경우.
   - 비밀번호 입력 필드에서 복사/붙여넣기 기능이 비활성화되어 있는 경우 (인지 부하 증가).
2. **검토 필요 (Review) [Default State for detection]:**
   - 페이지 내에 인지 기능 테스트가 필요한 인증 요소가 탐지되었으나, 대체 수단의 유효성이나 충분한 제공 여부를 기계적으로 판별하기 어려울 때 전문가의 확인이 필요한 상태.
3. **적절 (Pass):**
   - 인지 테스트가 필요한 인증 과정이 있더라도, 인지 테스트에 의존하지 않는 최소 한 가지 이상의 대체 인증 수단이 명확하게 제공됨이 확인된 경우.
4. **N/A:**
   - 페이지 내에 사용자 인증 과정이 존재하지 않음.
