# ABT-3.3.4-Processor: 반복 입력 정보 v0.10

### 🔗 References
- KWCAG 2.2: 3.3.4 반복 입력 정보
- WCAG 2.2: 3.3.8 Redundant Entry (A) (Reference)

---

이 알고리즘은 **KWCAG 7.3.4(반복 입력 정보)** 지침을 준수하며, 사용자가 동일한 프로세스(예: 회원가입-주소 입력, 주문서 작성-배송지 입력) 내에서 이전에 입력했던 정보를 다시 입력해야 하는 경우, 자동 입력(Autofill) 또는 선택 입력(Select from previous)과 같은 편리한 방법을 제공하는지 진단한다.
> 💡 **단일 페이지 검사 원칙 및 ABT 엔진 역할**
> ABT 엔진(브라우저 확장 프로그램)은 현재 로드된 단일 페이지의 DOM만을 분석하는 것을 원칙으로 한다. 따라서 여러 페이지에 걸친 "동일 프로세스" 내 반복 입력 필드의 실제 재입력 여부 및 대체 수단 제공 여부를 엔진이 자동으로 판단하는 것은 불가능하며, **탐지된 반복 가능 필드에 대해 전문가의 수동 검사를 강하게 유도**하는 방향으로 동작한다.

## 1. 대상 식별 및 데이터 수집 (ABT 엔진 역할)

### [대상 요소 추출]
- **데이터 입력 필드**: `<input>`, `<textarea>`, `<select>` 등 사용자로부터 정보를 받는 모든 필드.
- **관련 컨텍스트**: 필드 이름, `id`, `name`, `aria-label` 등을 기반으로 현재 페이지에서 이전에 입력되었을 가능성이 있는 필드.
- **속성**: `autocomplete` 속성 존재 및 값.
- **UI 요소**: "이전 주소와 동일", "회원 정보와 동일" 등의 체크박스 또는 버튼.

## 2. 분석 및 분류 로직 (단일 페이지 분석 및 전문가 가이드)

### [단계 A] 반복 가능 입력 필드 탐지
- **Rule 1.1 (반복 가능 필드 식별):** 현재 페이지의 입력 필드가 `name`, `id`, `aria-label` 등의 속성값을 통해 다른 페이지/폼에서 이전에 입력되었을 가능성이 있는 필드(예: `address-line1`, `email`, `name`)와 일치하는가?
  - **ABT 엔진 역할**: 현재 페이지 내에서 잠재적 반복 필드를 식별하여 전문가에게 제시.
  - 결과: 반복 가능성이 있는 필드가 탐지되면 **[검토 필요]** 상태로 전환.

### [단계 B] 자동 완성(Autofill) 또는 선택 입력 지원 검사 (단일 페이지 기준)
- **Rule 2.1 (`autocomplete` 속성):** 반복 가능성이 있는 필드에 `autocomplete` 속성이 `on` 또는 의미 있는 값(예: `address-line1`, `email`, `name`)으로 설정되어 있는가?
  - 결과: `autocomplete` 속성이 없거나 `off`로 설정된 경우 **[부적절]**.
- **Rule 2.2 (UI 기반 선택 입력):** "이전 정보와 동일" 등의 체크박스나 버튼이 현재 페이지에 명확하게 제공되는가?
  - 결과: 이러한 UI가 명확히 감지되지 않으면 **[부적절]** (특히 `autocomplete`도 없는 경우).

### [단계 C] 보안 관련 예외 처리
- **Rule 3.1 (보안 민감 정보):** 비밀번호, 카드 번호 등 보안상 필수적으로 재입력을 요구하는 필드인가?
  - 결과: 보안 민감 정보 필드는 **[해당 없음(예외)]** 처리.

### [단계 D] 전문가 수동 검사 가이드라인
도구는 전문가에게 다음 체크리스트를 제공하여, "동일 프로세스" 내에서 반복 입력이 발생하는 시나리오를 직접 수행하고, 자동/선택 입력 기능이 올바르게 작동하는지 수동으로 검증하도록 유도한다.
- *체크리스트*:
  - **반복 입력 시나리오 수행**: 이전 페이지/단계에서 특정 정보를 입력한 후, 현재 페이지에서 동일한 정보의 재입력을 요구하는가?
  - **자동 완성 기능 확인**: 브라우저의 자동 완성 기능 또는 `autocomplete` 속성에 의해 정보가 자동으로 채워지는가?
  - **선택 입력 UI 확인**: "이전 정보와 동일" 등의 UI가 클릭 시 정보를 채워주는가?
  - **보안 예외 확인**: 비밀번호 등 보안상 반복 입력이 필요한 필드였는가?

## 3. 최종 상태 정의 (Final Status)
1. **부적절 (Warning):**
   - 현재 페이지에서 `autocomplete` 속성이 없거나 `off`로 설정되어 있고, UI 기반의 선택 입력 수단도 제공되지 않는 필드가 발견된 상태에서, 전문가가 수동 검사 결과 "동일 프로세스 내 반복 입력"이 발생하고 있음을 확인한 경우.
2. **검토 필요 (Review) [Default State for detection]:**
   - ABT 엔진이 현재 페이지에서 반복 입력 가능성이 있는 필드를 탐지했으나, "동일 프로세스" 내에서의 실제 반복 여부, 자동 완성 기능의 유효성, UI 기반 선택 입력 수단의 작동 여부 등을 전문가가 직접 확인해야 하는 상태.
3. **적절 (Pass):**
   - 전문가가 수동 검사한 결과, "동일 프로세스 내 반복 입력"이 발생할 수 있는 필드에 `autocomplete` 속성이 적절히 적용되었거나, 명확한 UI 기반 선택 입력 수단이 제공됨이 확인된 경우.
4. **N/A:**
   - 현재 페이지에 반복 입력 정보가 필요한 상황이 없거나, 보안상 필수적으로 반복 입력이 요구되는 필드인 경우.
