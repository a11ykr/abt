1. 인식의 용이성 (Perceivable)
1. 적절한 대체 텍스트 제공
• 오류 판정 기준:
    ◦ <img> 등에 alt 속성이 없거나, 내용이 이미지와 다르거나(오타 포함), 불충분한 경우.
    ◦ QR코드 등 시각 정보만 제공하고 링크나 설명을 텍스트로 제공하지 않은 경우.
    ◦ 사용자가 등록하는 이미지에 대체 텍스트 입력 기능을 제공하지 않은 경우.
• 예외 처리 및 준수 인정:
    ◦ 장식용: 의미 없는 이미지(불릿 등)에 alt=""를 제공한 경우.
    ◦ 본문 설명: 본문에 이미지에 대한 충분한 설명이 있으면 alt=""로 제공해도 인정.
    ◦ 캡차(CAPTCHA): 이미지의 목적(예: "보안문자")만 대체 텍스트로 제공하고, 음성 등 대체 수단을 별도로 제공하면 인정.
    ◦ IR 기법: 적절한 내용을 숨김 텍스트로 제공한 경우 인정.
2. 자막 제공
• 오류 판정 기준:
    ◦ 멀티미디어에 자막, 대본, 수어 중 하나 이상을 제공하지 않거나, 내용이 동기화되지 않은 경우.
• 예외 처리 및 준수 인정:
    ◦ 파일 제공: 영상/음성 파일과 함께 자막 파일을 별도로 제공해도 인정.
    ◦ 단순 영상: 음성이 없는 동영상이라도 내용을 설명하는 대체 수단을 제공해야 함.
3. 표의 구성
• 오류 판정 기준:
    ◦ 데이터 테이블에 제목(<caption>)과 요약 정보가 없거나 부적절한 경우.
    ◦ 제목 셀(<th>)과 내용 셀(<td>)을 구분하지 않거나 연결(scope, headers)이 잘못된 경우.
    ◦ 배치용 테이블에 <th>, <caption>, summary를 사용한 경우.
• 예외 처리 및 준수 인정:
    ◦ 배치용 테이블: 선형화했을 때 이해 가능하면 배치용으로 간주.
    ◦ HTML5에서는 summary 속성을 사용하지 않으므로 <caption> 내에 요약 정보를 포함하면 인정.
4. 콘텐츠의 선형 구조
• 오류 판정 기준:
    ◦ CSS 제거 시 콘텐츠 순서가 논리적이지 않아 맥락 이해가 불가능한 경우 (예: 로그인 버튼이 아이디 입력보다 먼저 나옴).
    ◦ 계층 구조(메뉴 등)에 적절한 마크업(<ul>, <ol>, <hx> 등)을 사용하지 않은 경우.
• 예외 처리 및 준수 인정:
    ◦ 제목과 내용 사이에 관련 없는 콘텐츠가 있어도, 논리적으로 인식이 가능하면 인정.
5. 명확한 지시사항 제공
• 오류 판정 기준:
    ◦ 모양, 크기, 방향, 색, 소리 등 특정 감각에만 의존하여 지시하는 경우.
• 예외 처리 및 준수 인정:
    ◦ 특수 목적: 시각/청각 테스트, 감각 의존 게임 등은 심사 제외.
6. 색에 무관한 콘텐츠 인식
• 오류 판정 기준:
    ◦ 그래프, 차트, 필수 입력 항목 등을 색상만으로 구분하는 경우.
• 예외 처리 및 준수 인정:
    ◦ 중복 표시: 색상 외에 패턴, 굵기, 모양, 테두리, 글자 모양(밑줄, 이탤릭) 등으로 구분되면 인정.
    ◦ 대체 콘텐츠: 차트 내용을 텍스트(표 등)로 함께 제공하면 인정.
7. 자동 재생 금지
• 오류 판정 기준:
    ◦ 페이지 진입 시 3초 이상 자동으로 소리가 재생되는 경우.
    ◦ 마우스 오버나 초점 이동만으로 소리가 재생되는 경우.
• 예외 처리 및 준수 인정:
    ◦ 제어 기능: 3초 이상 재생되더라도 페이지 최상단에 정지/음소거 버튼이 있으면 인정.
8. 텍스트 콘텐츠의 명도 대비
• 오류 판정 기준:
    ◦ 일반 텍스트(18pt 미만): 4.5:1 미만.
    ◦ 큰 텍스트(18pt 이상 또는 굵은 14pt 이상): 3:1 미만.
• 예외 처리 및 준수 인정:
    ◦ 픽셀 기준: 윈도우 기준 14pt=18.66px, 18pt=24px로 적용.
    ◦ 장식/비활성: 장식 목적, 로고, 비활성화(disabled) 된 컨트롤은 제외.
9. 콘텐츠 간의 구분
• 오류 판정 기준:
    ◦ 이웃한 콘텐츠를 테두리, 구분선, 배경색, 간격 등으로 시각적으로 구분하지 않은 경우.

2. 운용의 용이성 (Operable)
10. 키보드 사용 보장
• 오류 판정 기준:
    ◦ 마우스로 가능한 기능을 키보드로 수행할 수 없는 경우.
• 예외 처리 및 준수 인정:
    ◦ 기술적 제약: GIS(지도 조작), VR, 붓칠, 3D 모델링 등 키보드 조작이 어려운 기능은 심사 제외 (단, 대체 수단 제공 시 인정).
11. 초점 이동과 표시
• 오류 판정 기준:
    ◦ 초점 이동 순서가 비논리적이거나 시각적으로 초점이 보이지 않는 경우.
    ◦ 초점이 갇히는 경우(Keyboard Trap).
• 예외 처리 및 준수 인정:
    ◦ 레이어 팝업: 팝업이 닫힐 때 초점이 원래 위치로 복귀하면 인정.
    ◦ 모바일: 화면낭독프로그램 초점(Accessible Focus) 순서가 일관되면 인정.
12. 조작 가능
• 오류 판정 기준:
    ◦ 컨트롤 크기가 대각선 6mm 미만인 경우.
    ◦ 컨트롤 안쪽 여백이 1px 미만인 경우.
• 예외 처리 및 준수 인정:
    ◦ 픽셀 환산: PC 웹 17px × 17px, 모바일 웹 24px × 24px 이상이면 준수 인정.
13. 문자 단축키
• 오류 판정 기준:
    ◦ 단일 문자 단축키를 제공하면서 끄거나 재설정하는 기능을 제공하지 않은 경우.
• 예외 처리 및 준수 인정:
    ◦ 초점 상태: 입력 창 등에 초점이 있을 때만 단축키가 동작하면 인정.
14. 응답시간 조절
• 오류 판정 기준:
    ◦ 시간 제한이 있는 콘텐츠에 연장/정지 수단을 제공하지 않은 경우.
• 예외 처리 및 준수 인정:
    ◦ 필수적 제한: 경매, 실시간 게임, 시험 등은 심사 제외.
    ◦ 20초 전 알림: 만료 20초 전에 연장 수단(간단한 키 조작)을 제공하면 인정.
15. 정지 기능 제공
• 오류 판정 기준:
    ◦ 자동으로 움직이거나 갱신되는 콘텐츠(롤링 배너 등)에 정지/이전/다음 기능이 없는 경우.
• 예외 처리 및 준수 인정:
    ◦ 대체 수단: "전체 보기" 등 전체 내용을 확인할 수 있는 목록을 제공하면 인정.
16. 깜빡임과 번쩍임 사용 제한
• 오류 판정 기준:
    ◦ 초당 3~50회 주기로 깜빡이는 콘텐츠를 사전 경고 없이 제공하는 경우.
17. 반복 영역 건너뛰기
• 오류 판정 기준:
    ◦ 건너뛰기 링크(본문 바로가기)가 없거나, 동작하지 않거나, 최상단에 없는 경우,.
• 예외 처리 및 준수 인정:
    ◦ 숨김 처리: 평소엔 안 보이다가 포커스 시 나타나면 인정.
18. 제목 제공
• 오류 판정 기준:
    ◦ 페이지, 프레임, 콘텐츠 블록(팝업 등)에 적절한 제목을 제공하지 않은 경우.
• 예외 처리 및 준수 인정:
    ◦ 내용 없는 프레임: "빈 프레임" 등의 제목 제공 시 인정.
    ◦ 단일 탭: 탭 메뉴가 사실상 1페이지인 경우 상위 범주 제목 허용.
19. 적절한 링크 텍스트
• 오류 판정 기준:
    ◦ 목적이나 용도를 알기 어려운 링크 텍스트(예: 빈 링크)를 제공한 경우.
• 예외 처리 및 준수 인정:
    ◦ 문맥 이해: "더보기", URL만 있어도 주변 문맥으로 목적 이해가 가능하면 인정.
20. 고정된 참조 위치 정보 (전자출판문서)
• 오류 판정 기준:
    ◦ e-Book 등에서 페이지 이동 기능이나 위치 정보가 없는 경우.
• 예외 처리 및 준수 인정:
    ◦ 고정 포맷: PDF와 같이 텍스트 스타일 변경이 불가한 문서는 점검 대상 아님.
21. 단일 포인터 입력 지원
• 오류 판정 기준:
    ◦ 다중 포인터(핀치 줌)나 경로 기반 동작(슬라이드)만 지원하고, 단일 포인터(클릭/탭) 대체 수단이 없는 경우.
• 예외 처리 및 준수 인정:
    ◦ 필수 동작: 서명, 피아노 건반 등 경로/다중 입력이 필수인 기능은 심사 제외.
    ◦ 브라우저 기본: 브라우저 자체 기능(뒤로 가기 스와이프 등)은 제외.
22. 포인터 입력 취소
• 오류 판정 기준:
    ◦ 누르는 동작(Down-Event)만으로 기능이 실행되어 취소할 수 없는 경우.
• 예외 처리 및 준수 인정:
    ◦ 취소 방법: 누른 상태에서 영역 밖으로 이동하여 떼면 취소되거나, 실행 후 되돌리기 기능이 있으면 인정.
    ◦ 필수 기능: 피아노 연주, 슈팅 게임 등은 심사 제외.
23. 레이블과 네임
• 오류 판정 기준:
    ◦ 시각적 텍스트 레이블이 접근 가능한 이름(Name)에 포함되지 않은 경우.
• 예외 처리 및 준수 인정:
    ◦ 심볼/수식: 기호(B, I, ABC)나 수식은 그 기능이나 수식 기호 그대로를 네임으로 제공하면 인정.
24. 동작 기반 작동
• 오류 판정 기준:
    ◦ 흔들기 등 동작(Motion)으로만 작동하고, 비활성화하거나 버튼으로 대체할 수 없는 경우.
• 예외 처리 및 준수 인정:
    ◦ 필수 동작: 만보계 등 동작이 필수인 기능은 심사 제외.

3. 이해의 용이성 (Understandable)
25. 기본 언어 표시
• 오류 판정 기준:
    ◦ <html>에 lang 속성이 없거나 잘못 기재된 경우.
26. 사용자 요구에 따른 실행
• 오류 판정 기준:
    ◦ 의도치 않게 새 창이 열리거나 초점이 변경되는 경우.
    ◦ <a target="_blank">로만 새 창을 알린 경우.
• 예외 처리 및 준수 인정:
    ◦ 기능적 필요: 문맥을 해치지 않는 범위에서 내용 갱신은 오류 아님.
27. 찾기 쉬운 도움 정보
• 오류 판정 기준:
    ◦ 각 페이지에서 도움 정보(연락처, 챗봇, FAQ 등)의 위치나 순서가 일관되지 않은 경우.
28. 오류 정정
• 오류 판정 기준:
    ◦ 입력 오류 시 원인을 알려주지 않거나 정정 수단이 없는 경우.
• 예외 처리 및 준수 인정:
    ◦ 보안/검증: 보안 관련 정보나 외부 시스템 검증 값은 예외 (단, 입력 중 오류는 정정 가능해야 함).
29. 레이블 제공
• 오류 판정 기준:
    ◦ 입력 서식에 <label> 또는 title을 제공하지 않은 경우.
• 예외 처리 및 준수 인정:
    ◦ 특정 타입: image, hidden, submit, button, reset 타입은 심사 제외.
30. 접근 가능한 인증
• 오류 판정 기준:
    ◦ 인지 기능 테스트(비밀번호 기억, 캡차, 퍼즐 등)만으로 인증해야 하는 경우.
• 예외 처리 및 준수 인정:
    ◦ 대체 수단: 생체 인증, USB 토큰, 기기 인증, 복사/붙여넣기 허용, 개인정보(이름/전화번호) 확인 등을 제공하면 인정.
31. 반복 입력 정보
• 오류 판정 기준:
    ◦ 동일 프로세스 내에서 이미 입력한 정보를 다시 입력하게 하는 경우.
• 예외 처리 및 준수 인정:
    ◦ 보안: 비밀번호 재입력 등 보안상 필수적인 경우는 심사 제외.

4. 견고성 (Robust)
32. 마크업 오류 방지
• 오류 판정 기준:
    ◦ 태그 열고 닫음 오류, 중첩 오류, 속성 중복 선언, ID 중복이 있는 경우.
33. 웹 애플리케이션 접근성 준수
• 오류 판정 기준:
    ◦ 웹 애플리케이션이 자체 접근성을 갖추지 못했고 대체 콘텐츠도 없는 경우.
